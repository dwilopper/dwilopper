Тут главное понять одну вещь: pgAdmin нужен не для запуска сайта. Он нужен только чтобы один раз создать базу PostgreSQL. Сам сайт потом запускается через Django из папки проекта.
Делай на другом компе вот так, прямо по порядку:
Установи:Python 3.12+
PostgreSQL
pgAdmin 4

Скачай проект с GitHub и распакуй его в обычную папку.

Открой pgAdmin 4 и подключись к своему локальному серверу PostgreSQL.

Создай пользователя и базу.
Самый быстрый способ: в pgAdmin открой Query Tool на базе postgres и выполни:

DO $$
BEGIN
    IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'buildmart_user') THEN
        CREATE ROLE buildmart_user WITH LOGIN PASSWORD 'buildmart_password';
    END IF;
END
$$;
Потом отдельно выполни:
CREATE DATABASE buildmart_store
OWNER buildmart_user;
Если база уже есть, этот второй запрос просто не нужен.
Закрой pgAdmin или оставь открытым, это уже не важно.

В папке проекта найди SETUP_LOCAL.bat и запусти его двойным кликом.
Он сам:
создаст .env
создаст .venv
установит зависимости
сделает migrate
загрузит стартовые данные

Если у тебя PostgreSQL работает не на порту 5432, перед запуском SETUP_LOCAL.bat открой .env и поменяй:POSTGRES_PORT=5432
на свой порт.

После успешной настройки запусти RUN_LOCAL.bat

В pgAdmin это делается так:
Открой pgAdmin 4.
Слева найди Servers.
Если там уже есть сервер, обычно называется что-то вроде PostgreSQL 17 / PostgreSQL 16 / localhost, просто нажми на него.
Он попросит пароль от postgres пользователя. Введи тот пароль, который ты задавал при установке PostgreSQL.
После этого сервер раскроется, и внутри будут:Databases
Login/Group Roles
остальное служебное

Если слева вообще пусто и сервера нет, тогда добавь его вручную:
ПКМ по Servers
Register
Server...
Дальше в окне заполни:
General
Name: например Local PostgreSQL
Connection
Host name/address: localhost
Port: 5432
Maintenance database: postgres
Username: postgres
Password: твой пароль, который задавал при установке PostgreSQL
Нажми Save
Если не подключается:
проверь, запущена ли служба PostgreSQL
проверь правильный ли пароль
проверь порт, иногда он не 5432
Если хочешь, я могу следующим сообщением дать тебе прям проверку:
“нажми вот сюда, и если видишь Databases, значит всё ок”.

Куда жать:
Слева нажми правой кнопкой на Servers
Выбери Register
Потом Server...
Дальше откроется окно. Заполни так:
General
Name: Local PostgreSQL
Connection
Host name/address: localhost
Port: 5432
Maintenance database: postgres
Username: postgres
Password: пароль, который ты задавал при установке PostgreSQL
можешь поставить галочку Save password
Потом нажми Save.
Если всё ок, слева появится сервер, и его можно будет раскрыть.
Если вдруг не подключится:
значит либо неверный пароль
либо у тебя PostgreSQL на другом порту
либо сам сервер PostgreSQL не запущен
Если хочешь, делай это сейчас, а потом скинь следующий скрин именно окна Register - Server, и я скажу, что куда вписывать по полям.
